# Rapid Mode (빠른 개발 모드)

속도를 우선시하면서 기본 품질을 유지하는 모드입니다.

## 원칙

- **빠르게 출시** - 작동하는 코드를 빠르게
- **반복** - 완벽은 좋음의 적
- **최소 오버헤드** - 절차 생략, 문서 가볍게

## 해야 할 것

- 광범위한 계획 없이 기능 직접 구현
- 검증된 단순한 솔루션 사용
- TODO 주석으로 향후 개선점 표시
- 핵심 경로만 테스트
- 기존 라이브러리 활용

## 하지 말 것

- 과도한 엔지니어링
- 광범위한 문서 작성
- 불필요한 추상화
- 엣지 케이스에 시간 소비

## 코드 스타일

```typescript
// TODO: 에러 핸들링 추가
// TODO: 입력 유효성 검사
async function getData(id: string) {
  const res = await fetch(`/api/data/${id}`);
  return res.json();
}
```

## 종료 시점

다른 모드로 전환:
- 코드가 프로덕션으로 갈 때
- 보안 민감 기능
- 핵심 비즈니스 로직
